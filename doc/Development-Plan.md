# 校园博客系统 —— 增量开发计划（更新版）

---

## 一、开发阶段总览

本项目采用 6 阶段增量开发，每个阶段产出可运行版本并可独立验收。在添加新功能时尽可能使用已有库和工具，避免重复造轮子。

```text
阶段 1 → 阶段 2 → 阶段 3 → 阶段 4 → 阶段 5 → 阶段 6
基础设施   内容模型    前台门户    认证与发布   互动与订阅   管理与上线
```

| 阶段 | 名称 | 核心交付物 | 预估工作量 |
|------|------|-----------|-----------|
| 阶段 1 | 基础设施与项目骨架 | 单项目可启动、D1/R2/KV 绑定可用 | 1 周 |
| 阶段 2 | 内容模型与后台管理 | 学校+子频道模型、后台管理可用 | 1.5 周 |
| 阶段 3 | 前端门户与内容展示 | 学校页/子频道页/文章页可浏览 | 1.5 周 |
| 阶段 4 | 认证与发布闭环 | 注册登录、发布、100MB 限额校验 | 2 周 |
| 阶段 5 | 互动与订阅 | 评论/点赞/收藏/关注 + 频道订阅 | 1.5 周 |
| 阶段 6 | 管理员运营与上线 | Review、配额管理、搜索、部署 | 1.5 周 |
| **合计** |  |  | **约 9 周** |

## 三、各阶段详细任务

### 阶段 1：基础设施与项目骨架

**目标**：完成当前模板基础设施接线，确保本地可直接开发。

**主要任务**：

1. 环境准备
- 安装 Node.js 24、Wrangler CLI
- `pnpm wrangler login`
- 创建 D1 / KV / R2 资源

2. 模板对齐
- 使用根目录 `wrangler.jsonc` 配置 D1、KV、R2 绑定
- 确认 `pnpm dev` 可启动 Next.js + Payload
- 确认 `/admin` 可访问

3. KV 策略定版
- 注册验证码、发送冷却、限流全走 KV
- **不实现 D1 降级方案**

4. 目录约定
- 业务 API：`src/app/api/`
- Payload 配置：`src/payload.config.ts`
- Drizzle 目录：`drizzle/`（阶段内创建）

1. 多语言基础设施
- 约定最小语言集：`zh-CN`、`en-US`
- 建立 i18n 资源目录与读取工具（禁止组件硬编码文案）

**阶段验收标准**：

- [√] `pnpm dev` 启动成功
- [√] `/admin` 可访问
- [√] `wrangler d1 execute D1 --local --command="SELECT 1"` 正常
- [√] 本地 `.wrangler/state/` 生成且 D1/KV/R2 可访问
- [√] 前台 i18n 基础设施可读取 `zh-CN` 与 `en-US` 词条

---

### 阶段 2：内容模型与后台管理

**目标**：完成两级频道模型与后台可维护能力。

**主要任务**：

1. 定义 Collection
- `School`（学校）
- `SchoolSubChannel`（学校子频道，归属 School）
- `Post`（schoolId 必填，subChannelId 可选）
- `Tag`、`Media`、`UserProfile`

2. 后台管理能力
- 管理员在 `/admin` 可新增/编辑/启用/排序学校与子频道
- 子频道归属关系校验

3. 种子数据
- 至少 3 所学校，每所学校至少 3 个子频道
- 至少 20 篇文章

**阶段验收标准**：

- [ ] 后台可完整管理学校与子频道
- [ ] Post 可正确绑定学校与子频道
- [ ] `hidden` 文章仅管理员可见

---

### 阶段 3：前端门户与内容展示

**目标**：完成公开浏览闭环（无需登录）。

**主要任务**：

1. 页面实现
- 首页：学校入口与推荐内容
- 学校页：`/school/[slug]`
- 子频道页：`/school/[slug]/channel/[channelSlug]`
- 文章页：`/post/[slug]`

2. 聚合逻辑
- 学校页展示该学校全部公开文章
- 子频道页展示该子频道公开文章
- 学校页支持按子频道筛选

3. UI 规范落地
- 采用现代化通透风格
- 响应式适配（桌面/移动）
- 首页、学校页、子频道页、文章页完成中英双语文案接入

**阶段验收标准**：

- [ ] 学校页与子频道页路由可访问
- [ ] 内容聚合与筛选逻辑正确
- [ ] `hidden` 内容不在前台出现
- [ ] 至少支持 `zh-CN` / `en-US` 切换，核心页面文案双语完整

---

### 阶段 4：认证体系与发布限额

**目标**：完成认证与发布能力，并启用 100MB 用户配额限制。

**主要任务**：

1. 认证接口
- `POST /api/auth/send-code`（KV 写验证码）
- `POST /api/auth/register`（KV 验证 + 建号）
- `POST /api/auth/login`
- `POST /api/auth/logout`

2. 登录态
- JWT 存储为 httpOnly Cookie
- `src/middleware.ts` 注入 `x-user-id`

3. 发布与配额
- 默认配额：`104857600` bytes
- 发布前计算“文字 + 图片”增量
- 超限拒绝并返回剩余额度
- 文章删除采用硬删除并立即回收配额（无恢复流程）

4. 用户中心
- `/user/me` 展示草稿、已发布、配额使用情况

**阶段验收标准**：

- [ ] 注册验证码流程全链路可用（仅 KV）
- [ ] 用户可登录并发布文章
- [ ] 超过 100MB 时发布被拒绝
- [ ] 硬删除文章后用户已用配额立即下降
- [ ] 用户中心可查看已用/总配额

---

### 阶段 5：互动与订阅

**目标**：补齐互动能力并上线频道订阅系统。

**主要任务**：

1. 互动功能
- 评论、点赞、收藏、关注

2. 订阅功能
- 学校订阅：`POST/DELETE /api/subscriptions/schools`
- 子频道订阅：`POST/DELETE /api/subscriptions/channels`
- 查询我的订阅列表：`GET /api/subscriptions/me`

3. 前端呈现
- 学校页“订阅学校”按钮
- 子频道列表“订阅子频道”开关

**阶段验收标准**：

- [ ] 互动功能状态可持久化
- [ ] 用户可订阅/取消订阅学校和子频道
- [ ] 取消学校订阅可级联清理该校子频道订阅

---

### 阶段 6：管理员运营与上线

**目标**：完成后台运营能力、生产部署与性能验收。

**主要任务**：

1. 内容运营
- 管理员 Review：`published` ↔ `hidden`

2. 配额运营（新增）
- 单用户配额调整
- 全量用户配额批量更新

3. 搜索与统计
- 搜索页（标题/摘要/标签）
- 阅读量统计（KV 热数据 + 定期回写 D1）

4. 生产部署
- Cloudflare 控制台绑定 D1/KV/R2
- `pnpm run deploy`

**阶段验收标准**：

- [ ] 管理员可在后台完成单用户与全量配额调整
- [ ] 搜索结果与筛选可用
- [ ] 生产部署成功并可访问核心页面

## 四、路径与工程约定（修正版）

- 绑定配置文件：`wrangler.jsonc`（根目录）
- 前台路由：`src/app/(frontend)`
- Payload 管理端与 API：`src/app/(payload)`
- 业务 Route Handlers：`src/app/api`（后续创建）
- Payload 配置：`src/payload.config.ts`
- 迁移目录：`src/migrations`
- Drizzle 目录：`drizzle/schema.ts` + `drizzle/migrations`

---

## 五、关键技术约束

1. KV 直接使用，不做验证码 D1 降级。
2. 发布配额默认 100MB，按字节严格校验（文字+图片）。
3. 管理员体系仅使用 Payload Users，与 `biz_users` 分离。
4. 项目开发期即遵循 i18n 规范：至少 `zh-CN` + `en-US`，新功能必须双语同步交付。
