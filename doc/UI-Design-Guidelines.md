# 校园博客系统 —— UI 设计指南

> 适用范围：`src/app/(frontend)` 前台站点。
> 管理端基于 Payload Admin（`src/app/(payload)`），仅增加必要运营功能界面。

---

## 1. UI 目标

1. 清晰表达两级频道结构：**学校 + 子频道**。
2. 让用户快速完成订阅、浏览、发布三个核心动作。
3. 在视觉上采用现代化通透风格，同时保证可读性与性能。
4. 在发布流程中清晰展示 100MB 配额与使用情况。

---

## 2. 技术框架与版本

前台 UI 与内容编辑使用以下技术：

1. Next.js `15.4.11`
2. shadcn/ui（CLI `4.1.0`）
3. Tailwind CSS `3.4.19`
4. Aceternity UI（组件拷贝式，按项目内引入）
5. Tiptap `3.20.4`
6. Payload CMS `3.77.0`

优先使用Aceternity UI，其次使用shadcn/ui预设风格

## 4. 视觉语言

### 4.1 风格关键词

`可信`、`清晰`、`现代化通透`、`校园感`

### 4.2 现代化通透风格（强制）

1. 导航、筛选条、浮层卡片允许半透明底色（例如 `bg-white/65`）。
2. 通透层需配合 `backdrop-blur`（`8px`~`16px`）和 1px 低对比边框。
3. 正文阅读区保持高可读性，禁用重度透明背景。
4. 动效数量受控，避免通透 + 大动效叠加导致卡顿。

### 4.3 颜色建议

- Primary：`#0D3B66`
- Secondary：`#1D7874`
- Accent：`#F4A259`
- Danger：`#D64545`
- Surface：`#F8FAFC / #FFFFFF / #EEF2F7`

---

## 5. 信息架构（两级频道）

### 5.1 导航结构

顶部导航建议包含：

1. Logo
2. 学校入口
3. 子频道入口（随学校切换）
4. 搜索
5. 写作入口
6. 登录/用户入口

### 5.2 路由建议

1. 首页：`/`
2. 学校页：`/school/[slug]`
3. 子频道页：`/school/[slug]/channel/[channelSlug]`
4. 文章页：`/post/[slug]`
5. 用户中心：`/user/me`
6. 写作页：`/editor`

### 5.3 可见性规则

1. 前台只展示 `published` 内容。
2. `hidden` 内容在列表、搜索、详情全部不可见。
3. 草稿只在作者自己可见。

---

## 6. 订阅交互规范

### 6.1 学校订阅

1. 学校页头部显示“订阅学校/已订阅”按钮。
2. 未登录点击订阅时，弹出登录引导。
3. 订阅成功后提供 Toast 与按钮状态更新。

### 6.2 子频道订阅

1. 学校页内子频道列表显示独立订阅开关。
2. 子频道订阅必须属于当前学校。
3. 取消学校订阅时，弹窗提示是否同步取消该校所有子频道订阅。

### 6.3 我的订阅

用户中心增加“我的订阅”：

1. 已订阅学校
2. 各学校下已订阅子频道
3. 快速取消入口

---

## 7. 发布配额 UI 规范（新增）

默认每用户总配额：`100MB`（文字 + 图片）。

### 7.1 写作页展示

1. 显示“已用 / 总配额”数字与进度条。
2. 实时显示本次草稿估算体积（文字字节 + 图片字节）。
3. 超限时禁用“发布”按钮并提示剩余可用容量。

### 7.2 用户中心展示

1. 配额卡片展示：已用、剩余、占比。
2. 当剩余 < 10% 时显示警告样式。

### 7.3 错误反馈

1. 超限提示文案需明确到字节或 MB。
2. 给出可执行建议：删除旧内容、压缩图片、联系管理员调额。

---

## 8. 页面规范

### 8.1 首页（`/`）

1. Hero + 学校卡片列表 + 推荐内容。
2. Hero 可使用 1 处 Aceternity 动效。
3. 学校卡片显示：名称、简介、子频道数量。

### 8.2 学校页（`/school/[slug]`）

1. 顶部：学校信息 + 订阅按钮。
2. 中部：子频道横向导航（含订阅开关）。
3. 下方：文章列表（支持“最新/热门”）。

### 8.3 子频道页（`/school/[slug]/channel/[channelSlug]`）

1. 显示子频道信息与订阅按钮。
2. 仅展示当前子频道文章。
3. 空状态文案聚焦该子频道。

### 8.4 文章详情页（`/post/[slug]`）

1. 标题区：标题、作者、发布时间、学校/子频道面包屑。
2. 正文区：Tiptap JSON 渲染后的 HTML。
3. 工具栏：点赞、收藏、分享。
4. 评论区：阶段化接入。

### 8.5 注册与登录页

1. 注册为两步同页切换：邮箱发码 → 验证码+资料。
2. 验证码冷却 60 秒倒计时。
3. 错误提示就近显示（字段下方优先）。

### 8.6 写作页（`/editor`）

1. 左侧编辑器，右侧元信息与配额卡片（移动端改上下布局）。
2. 元信息包含：学校、子频道、标签、封面。
3. 固定操作区：保存草稿、发布文章。

### 8.7 用户中心（`/user/me`）

1. 个人资料卡。
2. 内容管理：草稿 / 已发布 / 收藏。
3. 订阅管理：学校订阅 + 子频道订阅。
4. 配额卡片：已用/总额。

## 9. 组件规范

1. `components/ui/`：基础组件
2. `components/layout/`：导航、页脚、容器
3. `components/article/`：文章相关组件
4. `components/hierarchy/`：学校与子频道导航组件
5. `components/editor/`：编辑器与只读组件
6. `components/subscription/`（建议新增）：订阅按钮与订阅列表组件
7. `components/quota/`：配额进度条与超限提示组件

---

## 10. 状态与可用性

每个关键页面至少实现：

1. Loading（骨架屏）
2. Empty（可操作引导）
3. Error（可重试）
4. Success（明确反馈）

可访问性要求：

1. 对比度达到 WCAG AA。
2. 键盘可达，焦点可见。
3. 图片提供 `alt`。

## 12. 多语言 UI 开发规范

### 12.1 语言范围

1. 项目至少支持 `zh-CN` 与 `en-US`。
2. 新页面和新交互上线时必须同时提供中英文案。

### 12.2 文案与组件约束

1. 组件中禁止硬编码用户可见文案，统一走 i18n key。
2. 表单校验、错误提示、Toast、空状态文案全部纳入翻译资源。
3. 语言切换入口建议放在顶栏用户区附近，桌面与移动端都可达。

### 12.3 版式与可用性

1. 英文文案长度通常大于中文，按钮与标签需预留额外宽度。
2. 禁止依赖固定字符长度实现布局，优先自适应容器。
3. 日期、时间、数字、容量单位应按当前语言环境格式化显示。
